# Первая закупка в Китае — сайт-витрина

Статический сайт для просмотра карточек первой закупки в Китае (20 июня 2026 года).
Показывает список карточек с поиском и фильтром по статусу, а также подробную
карточку товара по клику.

Сайт состоит только из HTML/CSS/JS (без сборки, без фреймворков, без сервера) —
достаточно открыть `index.html` в браузере или раздать папку любым статическим
веб-сервером.

## Где лежат исходные ZIP-архивы

Исходные ZIP-архивы карточек закупки хранятся в папке `purchase-archives/`
(в корне репозитория, рядом с этой папкой `china-first-purchase/`). Один ZIP-архив —
одна карточка.

## Как добавить новую карточку

1. Положите ZIP-архив карточки в папку `purchase-archives`.
2. Запустите:
   ```
   python china-first-purchase/scripts/import_archives.py
   ```
3. Откройте:
   ```
   china-first-purchase/index.html
   ```

Импорт можно запускать повторно сколько угодно раз — он не создаёт дублей.
Если карточка с таким же номером уже была импортирована, она будет обновлена
(данные последнего обработанного архива побеждают).

## Как запустить импорт

Из корня репозитория:

```
python china-first-purchase/scripts/import_archives.py
```

Скрипт использует только стандартную библиотеку Python 3, дополнительные
зависимости не нужны.

Скрипт:
- читает все `*.zip` из `purchase-archives/`;
- в каждом архиве ищет ровно один `.md` файл с карточкой;
- извлекает из него структурированные данные (статус, наименование, цены,
  количество, поставщика и т.д.);
- копирует изображения и документы карточки в `assets/[номер карточки]/`;
- сохраняет оригинальный текст карточки в `cards/[номер карточки]/original-card.md`;
- пересобирает общий файл `data/purchases.json`, который читает сайт.

Если в архиве не найден `.md` файл, найдено больше одного `.md` файла, или не
удалось извлечь какое-то поле — скрипт выведет предупреждение в консоль, а сама
карточка (если её всё же удалось собрать) получит список предупреждений в поле
`warnings` в `purchases.json`; такие карточки на сайте дополнительно помечаются
значком предупреждения.

## Как открыть сайт

Это статический сайт, поэтому подойдёт любой способ раздачи статических файлов,
например:

```
cd china-first-purchase
python -m http.server 8000
```

и затем открыть `http://localhost:8000/index.html` в браузере.

Открытие `index.html` напрямую через `file://` может не работать в некоторых
браузерах из-за ограничений на `fetch()` локальных файлов — рекомендуется
использовать локальный веб-сервер, как показано выше.

## Структура папки

```
china-first-purchase/
  index.html              — страница сайта
  styles.css              — стили
  app.js                  — вся логика сайта (поиск, фильтры, рендер карточек)
  data/purchases.json      — собранные данные всех карточек (генерируется импортом)
  assets/[номер]/...       — изображения и документы карточки (генерируется импортом)
  cards/[номер]/original-card.md — оригинальный текст карточки (генерируется импортом)
  scripts/import_archives.py — скрипт импорта ZIP-архивов
```

## Какие папки/файлы можно менять

- `purchase-archives/` — добавлять новые ZIP-архивы карточек.
- Содержимое `china-first-purchase/data/`, `china-first-purchase/assets/`,
  `china-first-purchase/cards/` — полностью генерируется скриптом импорта,
  вручную их можно не трогать (но это безопасно, так как при повторном запуске
  импорта они пересоздаются).

## Какие папки/файлы нельзя трогать

- Любые файлы и папки репозитория за пределами `purchase-archives/` и
  `china-first-purchase/` — импорт и сайт их не используют и не изменяют.
- Оригинальные ZIP-архивы в `purchase-archives/` никогда не копируются на сайт и
  не становятся доступны для скачивания — наружу попадают только распакованные
  файлы и текст карточки.
